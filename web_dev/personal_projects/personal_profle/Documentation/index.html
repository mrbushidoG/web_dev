2. HTML Structure

This theme is a fixed layout with two columns. All of the information within the main content area is nested within a "main" element with an class of content. The sidebar's content is within a div with class name aside-container. The general template structure is the same throughout the template except welcome screen page. Here is the general structure.

3.2. Welcome screen

  1. Types of welcome screen page
    The welcome screen page can be displayed in two ways:
    - with a image in the background on the entire screen and centered text - default
    - with the screen split vertically in half, where in one part is displayed image, and the second is displayed text.
    To enable this type of welcome screen page you must add the class half-page to element with id welcome-screen By default image is displayed on the left side but you can set it to right by adding image-right class. Example:
    <!--- Page is split into two parts with a photo on the left --->
<div id="welcome-screen" class="half-page"> ... </div> 

<!--- Page is split into two parts with a photo on the right --->
<div id="welcome-screen" class="half-page image-right"> ... </div>
  2. How to change background image
    You can change the background image in a three ways. Image should have a size of 1920px / 900px.
    1. First and the easiest way is to overwrite the images/background.png and images/background@2x.png files with your images. Example:
    2. The second way is to change image path in CSS rule #welcome-screen .image.
    #welcome-screen .image { 
 background-image: url('../images/my-background.png');
}
/* For retina displays */
@media (-webkit-min-device-pixel-ratio: 1.5), (min--moz-device-pixel-ratio: 1.5), (-o-min-device-pixel-ratio: 3/2), (min-resolution: 1.5dppx) {
 #welcome-screen .image { 
  background-image: url('../images/my-background@2x.png');
 }
}​

    3. The last and not recommended method is to set in html code css rule background-image. Example:

    <div id="welcome-screen">
 <div class="image" style="background-image: url(/path/to/my-background.png)" data-rjs="2"></div>
 ...
</div>

7.2. Configure contact form

Contact form settings:

Required
  • host - SMTP server
  • username - SMTP username
  • password - SMTP password
Optional, default values
  • fromEmail - Sender e-mail address, if not set the username option will be used.
  • fromName - Sender name
  • toEmail - Receiver e-mail address, if not set the username option will be used.
  • toName - Receiver name
  • subject - Message subject. You can set here a form field value inserting in it braces ie. {field_name}.
  • messagePrefix - Content added before table with form data. You can set here a form field value inserting in it braces {field_name}.
  • messageSuffix - Content added after table with form data. You can set here a form field value inserting in it braces {field_name}.
  • encryption (Default: tls) - Enable TLS encryption, 'ssl' also accepted.
  • port (Default: 587) - CP port to connect to.
  • phpCertificateFix (Default: false) - For more information about it please see this page: https://github.com/PHPMailer/PHPMailer/wiki/Troubleshooting#php-56-certificate-verification-failure. Enabling this option is highly risky. If you have this problem firstly try to fix your SSL certificate.
  • debug - Enable debug mode with more detailed messages
  • confirmationMessage (Default: "Message was send.") - Message displayed to user when the form was sent successfully.
  • errorMessage (Default: "There was a problem while sendig a message, please try again later or contact with my by phone or skype.") - Message displayed to user when something goes wrong.
  • validateMessages (Default: Array) - Array of validator messages
  • validateMessages.required (Default: "This field is required.") - Message displayed when field is required
  • validateMessages.is_email (Default: "This is not a valid e-mail address.") - Message displayed when field must be a valid e-mail address
  • validateMessages.recaptcha - Message displayed if reCaptcha validation failed
  • recaptcha (Default: Array) - Array of reCaptcha settings
  • recaptcha.enabled (Default: false) - Enable reCaptcha
  • recaptcha.site_key - reCaptcha site key
  • recaptcha.secret_key - reCaptcha secret key
  • fields (Default: Array) - Array of form fields configuration
  • fields.example - Array of "example" field configuration
  • fields.example.label - Label of field replaced in message
  • fields.example.required - Required validation
  • fields.example.is_email - E-mail validation

The configuration file can be stored as php array or .ini files. Examples:

return array(
 // These three options must be set.

8.2. Edit & compilation

If you want to make some changes in the LESS files and re-compile it, you should compile only the files that are located directly in the directory /css /less. The files in the directory css/less/config and css/less/includes should not be compiled directly. If you want to create your own main LESS file, you should load into it configuration file from the config directory and /css/less/styles.less file. Example:

// Import default configuration file
@import "configs/default";

// Import theme strucure and styles
@import "include/styles";

// Additional styles
...

9. Grunt

If you do not know what it is Grunt before continuing see the information on this page:http://gruntjs.com.

9.1. Available tasks

List of available tasks:

  • uglify - combine and minimize JavaScript files
  • cssmin - combine and minimize CSS files
  • less - compile LESS files to CSS
  • watch - observed changes in LESS, CSS and JavaScript files and executes the appropriate task

You can configure tasks performed by the Grunt, to do this edit the file Gruntfile.js. At the beginning there are variables with simple tasks configuration, eg. a list of compressed CSS and JavaScript files, the name and location of the resulting files.
